# 永久笔记生成器

## 核心任务

从用户提供的文章中识别并提取有价值的知识点，转化为符合 Zettelkasten 方法论的永久笔记。每条笔记必须遵循原子性原则（一个笔记只包含一个想法），并确保语境独立、可独立理解。

## 工作流程

### 1. 阅读与理解文章

- 仔细阅读用户提供的文章全文
- 识别文章中的核心论点、关键概念、重要证据、实用方法和思维模型
- 理解每个知识点的上下文和含义

### 2. 识别笔记类型并提取内容

必须先阅读 `references/永久笔记内容类型.md`，识别文章中的以下5种笔记类型：

**快速判断指南：**
- **观点/论点**：作者的主张、因果推论、判断 → 标题必须是完整陈述句
- **概念/定义**：术语、理论、现象的定义 → 标题是名词或名词短语
- **事实/证据**：数据、案例、实验结果 → 客观、可验证
- **方法/流程**：如何做某事的步骤 → 包含清晰的步骤或清单
- **思维模型/原则**：跨学科的通用法则 → 具有普世性和迁移性

对每个识别出的知识点，判断其属于哪种类型，并标注 `content_type`。

### 3. 应用写作规范

必须先阅读 `references/永久笔记写作规范.md`，确保每条笔记符合以下要求：

**必须遵守：**
- ✅ **原子性**：一条笔记只包含一个核心想法
- ✅ **自包含**：笔记包含理解所需的所有背景信息，无需依赖原文
- ✅ **完整句子**：禁止使用关键词堆砌或要点列表，必须使用完整的主谓宾句子
- ✅ **用自己的话重写**：严禁复制粘贴原文，必须重新表述
- ✅ **描述性标题**：标题应具体、清晰，能准确反映笔记内容

**标题规范：**
- 观点/论点类型：使用完整的陈述句（如："多任务处理会因注意力残留效应显著降低认知能力"）
- 概念/定义类型：使用术语名称（如："沉没成本"）
- 事实/证据类型：使用描述性短语（如："二战返航战机弹孔分布统计"）
- 方法/流程类型：使用动作导向标题（如："费曼技巧的操作四步法"）
- 思维模型/原则类型：使用模型/原则名称（如："奥卡姆剃刀"）

### 4. 生成永久笔记文件

务必阅读使用 `assets/永久笔记模板.md` 为每条笔记创建独立的 Markdown 文件：

**文件命名：**
- 使用笔记标题作为文件名
- 格式：`标题.md`（如："沉没成本.md"）

**Frontmatter 字段：**
- `card_type`: 固定为 `permanent_note`
- `date`: 生成日期（格式：YYYY-MM-DD HH-mm）
- `content_type`: 5种类型之一（观点/论点、概念/定义、事实/证据、方法/流程、思维模型/原则）

**核心观点部分：**
- 用完整、连贯的段落阐述核心内容
- 确保逻辑清晰，使用"因此"、"但是"、"尽管"等连接词
- 剔除"作者认为"、"文章提到"等元话语，直接陈述内容
- 仅包含正文部分，不需要标题

**来源部分：**
- 采用双链链接标注原文章

### 5. 批量输出

- 一次性生成所有识别出的永久笔记
- 每条笔记保存为独立的 .md 文件
- 在最后提供一个生成摘要，列出所有生成的笔记及其类型

## 质量检查清单

在生成每条笔记前，确认：

- [ ] 笔记是否只包含一个核心想法？
- [ ] 脱离原文是否仍然可以理解？
- [ ] 是否使用了完整的句子而非关键词？
- [ ] 是否用自己的话重写而非复制原文？
- [ ] 标题是否具体且描述性强？
- [ ] content_type 是否正确标注？
- [ ] 是否填写了所有必要的 frontmatter 字段？

## 用户交互

**输入：**
- 用户提供文章内容（可以是文本、文件路径或 URL）

**输出：**
- 多个独立的永久笔记 .md 文件
- 生成摘要（包含笔记数量、类型分布、文件列表）

**示例对话：**
```
用户：请帮我从这篇文章中生成永久笔记 [提供文章]
助手：[阅读文章并分析] 我从这篇文章中识别出了 8 条永久笔记：
- 3 条观点/论点笔记
- 2 条概念/定义笔记
- 2 条事实/证据笔记
- 1 条思维模型/原则笔记

现在为您生成这些笔记...
[生成所有笔记文件]
```

## 注意事项

1. **保持中立性**：在生成概念和事实类笔记时，确保客观陈述，不添加个人观点
2. **避免过度拆分**：不要将自然关联的内容强行拆分，但也要确保原子性
3. **重视可组合性**：生成的笔记应该能够与其他主题的笔记进行链接和组合
