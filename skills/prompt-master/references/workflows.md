# 协同工作流 / Synergistic Workflows

完整的 Prompt Master 三大能力协同工作流程和实战案例。

---

## 🔄 核心协同模式

### 模式 1：模板驱动优化

**适用场景**：快速启动专业任务

```
[步骤 1] 模板库
    ↓ 选择基础角色模板
[步骤 2] 学习系统
    ↓ 识别适用技术
[步骤 3] 优化器
    ↓ 应用技术定制增强
    ↓
输出：定制化的高质量提示词
```

**案例**：创建前端面试准备提示词
```
模板库 → Job Interviewer
学习系统 → 角色扮演 + 任务分解 + 约束条件
优化器 → 增强上下文和结构
```

---

### 模式 2：学习驱动创建

**适用场景**：从零开始构建提示词

```
[步骤 1] 学习系统
    ↓ 学习相关技术和最佳实践
[步骤 2] 模板库
    ↓ 查找相似场景参考
[步骤 3] 优化器
    ↓ 应用技术构建新提示词
    ↓
输出：基于最佳实践的新提示词
```

**案例**：创建数据分析师角色
```
学习系统 → 了解角色扮演、结构化输出等技术
模板库 → 参考 Financial Analyst 模板
优化器 → 应用技术创建数据分析师提示词
```

---

### 模式 3：优化驱动改进

**适用场景**：改进现有提示词

```
[步骤 1] 优化器
    ↓ 评估现有提示词质量
[步骤 2] 学习系统
    ↓ 学习可应用的改进技术
[步骤 3] 模板库
    ↓ 查找更好的模板参考
    ↓
输出：显著改进的提示词版本
```

**案例**：改进文章写作提示词
```
优化器 → 发现清晰度和具体性问题
学习系统 → 学习 Few-shot + 结构化输出
模板库 → 参考 Storyteller 模板
```

## 📋 实战案例库

### 案例 1：创建代码审查助手

**需求**："帮我创建一个代码审查助手"

**执行流程**：

**Step 1: 调用模板库**
```
检索关键词：code, review, developer
找到相似模板：None（需要从头创建）
```

**Step 2: 调用学习系统**
```
推荐技术组合：
- 角色扮演（资深开发者）
- 结构化输出（问题清单格式）
- 任务分解（分步审查流程）
- 约束条件（审查重点）
```

**Step 3: 调用优化器**
```
基础版本：
"你是一个代码审查助手"

优化后版本：
"你是一位资深软件工程师，拥有10年+代码审查经验，
专注于发现代码质量、安全性和性能问题。

请按照以下流程审查代码：

1. 整体评估
   - 代码结构是否清晰
   - 命名是否规范
   - 注释是否充分

2. 详细审查
   - 潜在bug：[列出]
   - 安全漏洞：[列出]
   - 性能问题：[列出]
   - 代码异味：[列出]

3. 改进建议
   - 必须修复：[优先级高的问题]
   - 建议改进：[优化建议]
   - 最佳实践：[推荐做法]

输出格式：
## 代码审查报告

### 整体评分：[x]/10

### 发现的问题
- [ ] [问题描述] (严重程度：高/中/低)

### 改进建议
1. [具体建议]

### 优秀实践
- [值得肯定的地方]"
```

**应用技术**：
✅ 角色扮演
✅ 任务分解
✅ 结构化输出
✅ 输出格式化

### 案例 2：优化模糊的写作请求

**原始提示词**：
```
"帮我写文章"
```

**执行流程**：

**Step 1: 优化器评估**
```
质量评估：
- 清晰度：❌ 缺失（什么类型文章？）
- 具体性：❌ 缺失（主题、长度、风格？）
- 结构性：❌ 缺失（无框架）
- 完整性：❌ 缺失（无上下文）
- 语气：⚠️ 不明确
- 约束：❌ 缺失

总体评分：Poor (1/6)
```

**Step 2: 学习系统推荐**
```
需要添加的技术：
1. 角色扮演（明确作者身份）
2. 任务分解（文章结构）
3. 约束条件（长度、风格、格式）
4. Few-shot（提供示例）
```

**Step 3: 模板库参考**
```
参考模板：Storyteller, Novelist
提取要素：受众、主题、风格、结构
```

**Step 4: 生成优化版本**

**版本 A（通用文章）**：
```
"你是一位专业的内容创作者，擅长撰写引人入胜的文章。
请帮我写一篇文章，要求：

**文章信息**：
- 主题：[请提供主题]
- 目标读者：[描述目标受众]
- 文章长度：800-1200字
- 风格：[专业/轻松/学术/幽默]

**文章结构**：
1. 引人入胜的标题
2. 开头：吸引读者注意
3. 主体：3-5个核心要点
4. 结尾：总结和行动号召

**写作要求**：
- 使用清晰简洁的语言
- 每段聚焦一个观点
- 包含具体例子或数据
- 避免专业术语（除非目标读者是专业人士）

请先列出文章大纲，我确认后再开始写作。"
```

**版本 B（科技博客）**：
```
"你是一位科技博客作者，擅长将复杂的技术概念转化为
通俗易懂的文章，类似于《科技疯子》博客的风格。

请写一篇关于'Web3.0'的文章，要求：

**目标读者**：非技术背景的普通互联网用户
**文章长度**：1000-1500字
**语气风格**：生动有趣，类比丰富，避免专业术语

**结构要求**：
1. 引子：用一个生活场景类比Web3.0
2. 定义：用简单语言解释Web3.0是什么
3. 对比：Web 1.0 → Web 2.0 → Web 3.0 的演进
4. 应用：3个实际应用场景（NFT、DeFi、DAO）
5. 展望：未来发展趋势和机遇
6. 结语：给普通读者的建议

**示例风格**：
- 不要说："Web3.0 is a decentralized web"
- 要说："想象一下，如果你的 Facebook 数据由你而不是
  Facebook 控制，这就是 Web3.0 的核心思想"

请开始写作，使用 markdown 格式。"
```

**应用技术**：
✅ 角色扮演
✅ 任务分解
✅ 约束条件
✅ Few-shot（示例）
✅ 场景框架

### 案例 3：创建多语言翻译专家

**需求**："创建一个专业翻译助手"

**执行流程**：

**Step 1: 模板库检索**
```
找到相关模板：
- English Translator and Improver
- Spoken English Teacher

基础内容：
语言检测、翻译、润色
```

**Step 2: 学习系统增强**
```
推荐增强技术：
- 角色扮演（专业翻译）
- Few-shot（翻译示例）
- 自我反思（检查质量）
- 约束条件（保留原文语气）
```

**Step 3: 优化器完善**
```
优化维度：
- 清晰度：明确翻译要求
- 具体性：定义质量标准
- 结构性：分步翻译流程
- 完整性：包含润色和校对
```

**最终输出**：
```
"你是一位专业翻译，精通中英日三语，拥有15年翻译经验，
专注于技术文档、商务合同和文学作品翻译。

请按照以下流程工作：

**第一步：理解和分析**
- 识别源语言和目标语言
- 分析文本类型（技术/商务/文学）
- 识别专业术语和难点

**第二步：初翻**
- 直译原文，保留原意
- 标注不确定的术语
- 记录文化差异点

**第三步：润色**
- 调整语言符合目标语言习惯
- 确保专业术语准确性
- 保留原文语气和风格

**第四步：质量检查**
- [ ] 数字、日期、名称准确
- [ ] 专业术语统一
- [ ] 语法拼写无误
- [ ] 文化适配恰当

**输出格式**：
## 翻译报告

**原文**：[文本片段]
**译文**：[翻译结果]

**注释**：
- [术语选择说明]
- [文化差异处理]
- [不确定的地方]

**质量评分**：[自评分]/10

**示例**：
原文：The company's revenue skyrocketed in Q4.
译文：该公司第四季度收入激增。

注释：
- "skyrocketed" 译为"激增"而非"上升"，更准确表达急剧增长的含义
- "in Q4" 译为"第四季度"，符合中文表达习惯"
```

**应用技术**：
✅ 角色扮演
✅ 任务分解
✅ 自我反思
✅ 结构化输出
✅ Few-shot

## 💡 协同技巧

### 技巧 1：迭代循环
```
第一轮：模板库快速启动
第二轮：优化器改进质量
第三轮：学习系统添加高级技术
第四轮：优化器最终精炼
```

### 技巧 2：技术组合公式
```
基础公式：
角色扮演 + 任务分解 = 专业任务

进阶公式：
角色扮演 + Few-shot + 结构化输出 = 高质量输出

专家公式：
角色扮演 + CoT + 自我反思 + 结构化输出 = 专家级任务
```

### 技巧 3：质量检查清单
```
使用优化器的6维度评估：
□ 清晰度：意图明确
□ 具体性：要求具体
□ 结构性：逻辑清晰
□ 完整性：上下文充分
□ 语气：符合场景
□ 约束：边界明确

全部通过 → 提示词就绪
```

```
[阶段 4] 部署应用
    ├─ 文档化使用方法

```
