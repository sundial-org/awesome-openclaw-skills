## 🔄 协同工作流

### 完整流程示例

```
用户："帮我准备一个前端开发面试"

↓ [步骤 1] 模板库
获取 "Job Interviewer" 模板

↓ [步骤 2] 学习系统
推荐技术：角色扮演 + 任务分解 + 约束条件

↓ [步骤 3] 优化器
应用技术生成定制版本

↓ 输出
专业的面试准备提示词（包含5个环节）
```

### 三大协同模式

**模式1：模板驱动优化**
```
模板库 → 选择基础模板
学习系统 → 识别适用技术
优化器 → 定制增强
```

**模式2：学习驱动创建**
```
学习系统 → 学习相关技术
模板库 → 参考相似模板
优化器 → 构建新提示词
```

**模式3：优化驱动改进**
```
优化器 → 评估现有提示词
学习系统 → 学习改进技术
模板库 → 查找更好参考
```

## 🛠️ 安装和使用

### 方式 1：全局配置（推荐）

将 Prompt Master 添加到全局配置 `~/.claude/CLAUDE.md`：

```bash
# 自动配置
cat prompt-master/CLAUDE.md >> ~/.claude/CLAUDE.md
```

### 方式 2：项目级配置

将此技能复制到项目的 `.claude/` 目录：

```bash
cp -r prompt-master /your/project/.claude/skills/
```

### 方式 3：手动加载

在对话中明确引用：

```
"使用 prompt-master 技能帮我..."
"调用提示词模板库..."
"学习 Few-shot 技术..."
"优化这个提示词..."
```
