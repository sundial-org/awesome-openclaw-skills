# 优化示例：从糟糕到优秀

## 学习目标
通过真实案例，学习如何将糟糕的提示词优化为优秀的提示词

---

## 案例 1：文章写作

### ❌ 糟糕版本
```
"帮我写文章"
```

**质量评估**：
- 清晰度：❌ 什么文章？
- 具体性：❌ 无任何要求
- 结构性：❌ 无框架
- 完整性：❌ 无上下文
- 语气：❌ 不明确
- 约束：❌ 无约束

**评分**：0.5/6 - Poor

---

### ⚠️ 一般版本
```
"帮我写一篇关于AI的文章，1000字左右"
```

**质量评估**：
- 清晰度：✅ 明确主题
- 具体性：⚠️ 只有长度要求
- 结构性：❌ 无结构
- 完整性：⚠️ 缺少读者、风格
- 语气：❌ 不明确
- 约束：⚠️ 仅长度

**评分**：2/6 - Fair

---

### ✅ 良好版本
```
"你是一位科技博客作者。请写一篇关于'AI如何改变日常生活'的文章，
要求：1000字，面向普通读者，包含3个实际应用案例。"
```

**质量评估**：
- 清晰度：✅ 角色明确
- 具体性：✅ 有长度、读者、案例要求
- 结构性：⚠️ 结构不够详细
- 完整性：✅ 上下文充足
- 语气：✅ 科技博客风格
- 约束：✅ 有约束

**评分**：4/6 - Good

**应用技术**：
- ✅ 角色扮演
- ✅ 任务定义
- ✅ 约束条件

---

### ⭐ 优秀版本
```
"你是一位科技博客作者，风格类似于《科技疯子》博客，
擅长将复杂技术概念转化为通俗易懂的文章。

请写一篇关于'AI如何改变日常生活'的文章：

**目标读者**：非技术背景的普通互联网用户
**文章长度**：1000-1200字
**语气风格**：生动有趣，类比丰富，避免专业术语

**文章结构**：
1. 引人入胜的开头（用一个生活场景引入AI）
2. 三个实际应用案例：
   - 智能助手（如Siri、小爱同学）
   - 推荐系统（如抖音、淘宝推荐）
   - 图像识别（如人脸解锁、垃圾分类）
3. 每个案例包含：
   - 场景描述
   - AI如何工作（简单解释）
   - 对生活的影响
4. 未来展望（2-3个即将到来的AI应用）
5. 结语（给普通读者的建议）

**写作要求**：
- 使用类比解释技术概念
- 每段聚焦一个观点
- 包含具体数据和例子
- 避免专业术语，必要时用通俗语言解释

**示例风格**：
- 不要说："计算机视觉技术..."
- 要说："就像人眼识别物体一样，AI也能'看懂'图片..."

请先列出文章大纲，我确认后再开始写作。"
```

**质量评估**：
- 清晰度：⭐ 优秀
- 具体性：⭐ 优秀
- 结构性：⭐ 优秀（详细结构）
- 完整性：⭐ 优秀（完整上下文）
- 语气：⭐ 优秀（风格明确）
- 约束：⭐ 优秀（多维度约束）

**评分**：6/6 - Excellent

**应用技术**：
- ✅ 角色扮演（具体风格参考）
- ✅ Few-shot（示例风格）
- ✅ 任务分解（详细结构）
- ✅ 约束条件（多维度）
- ✅ 场景框架（目标读者）
- ✅ 输出格式化（先大纲后正文）

**改进效果**：
```
糟糕版本 → 模糊输出，质量不稳定
一般版本 → 有主题但结构混乱
良好版本 → 有结构但不够详细
优秀版本 → 专业级输出，质量稳定 ⭐
```

## 案例 2：代码调试

### ❌ 糟糕版本
```
"我的代码有bug，帮我看看"
```

**问题**：没有提供任何代码或错误信息

---

### ⚠️ 一般版本
```
"我的Python代码有bug，帮我找出来：

```python
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)
```
"
```

**问题**：缺少上下文，没有说明具体问题

---

### ✅ 良好版本
```
"我的Python函数应该计算平均值，但有时候会报错。

代码：
```python
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)
```

错误信息：ZeroDivisionError: division by zero

帮我找出问题并修复。"
```

**进步**：提供了代码和错误信息

---

### ⭐ 优秀版本
```
"你是一位资深Python调试专家。我需要你帮我调试一个函数。

**问题描述**：
函数应该计算数字列表的平均值，但在空列表时报错。

**代码**：
```python
def calculate_average(numbers):
    total = 0
    for num in numbers:
        total += num
    return total / len(numbers)

# 测试
print(calculate_average([1, 2, 3]))  # 正常
print(calculate_average([]))        # 报错
```

**错误信息**：
```
Traceback (most recent call last):
  File "test.py", line 7, in <print(calculate_average([]))>
  File "test.py", line 4, in <return total / len(numbers)>
ZeroDivisionError: division by zero
```

**期望输出**：
- 对于非空列表：返回平均值
- 对于空列表：返回0或None（你建议哪个更好？）

**请提供**：
1. 问题根因分析
2. 修复后的代码
3. 改进建议（如有其他潜在问题）
4. 单元测试示例

**输出格式**：
## 问题分析
[根因]

## 修复方案
```python
[修复后的代码]
```

## 改进建议
[你的建议]

## 测试代码
```python
[测试用例]
```"
```

**改进效果**：
```
糟糕 → 无信息，无法调试
一般 → 有代码，缺上下文
良好 → 有代码和错误，可调试
优秀 → 专业级调试流程 ⭐
```

**应用技术**：
- ✅ 角色扮演（资深Python专家）
- ✅ 场景框架（完整问题描述）
- ✅ 任务分解（4个输出要求）
- ✅ 结构化输出（markdown格式）
- ✅ Few-shot（隐含在格式要求中）

## 案例 3：邮件撰写

### ❌ 糟糕版本
```
"帮我写封邮件"
```

---

### ⚠️ 一般版本
```
"帮我写一封请假邮件，请3天假"
```

---

### ✅ 良好版本
```
"帮我写一封请假邮件给老板，因为家里有事要请3天假，
从下周一开始。"
```

---

### ⭐ 优秀版本
```
"你是一位专业的商务沟通顾问。我需要写一封请假邮件。

**背景信息**：
- 收件人：我的直属领导张总
- 请假原因：老家有急事需要处理
- 请假时间：下周一到周三（3天）
- 工作安排：已和同事李四交接，他会处理紧急事务
- 补充说明：保持手机畅通，有急事可联系

**邮件风格**：
- 语气：礼貌、专业、诚恳
- 格式：标准商务邮件
- 长度：简洁明了（200字以内）

**邮件结构**：
1. 邮件主题（明确、专业）
2. 称呼
3. 请假事由（简要说明）
4. 请假时间
5. 工作安排（已交接）
6. 保持联系
7. 致歉和感谢
8. 祝福语
9. 署名

**请提供两个版本**：
- 版本A：正式版（适合严格的企业文化）
- 版本B：稍微轻松版（适合开放的团队文化）

请用中文撰写。"
```

**应用技术**：
- ✅ 角色扮演（商务沟通顾问）
- ✅ 场景框架（完整背景）
- ✅ 任务分解（9点结构）
- ✅ 约束条件（字数、语气）
- ✅ Few-shot（要求两个版本）

## 📚 学习总结

### 核心要点
1. **糟糕的提示词** = 模糊、无约束、无结构
2. **一般的提示词** = 有任务、缺细节
3. **良好的提示词** = 有角色、有结构、有约束
4. **优秀的提示词** = 详细、结构化、有示例

### 优化公式
```
优秀提示词 = 角色定位 + 详细任务 + 具体约束
             + 输出格式 + Few-shot示例 + 质量标准
```

### 6维度评估
- 清晰度：意图是否明确
- 具体性：要求是否具体
- 结构性：是否有逻辑组织
- 完整性：上下文是否充分
- 语气：是否符合场景
- 约束：边界是否明确
